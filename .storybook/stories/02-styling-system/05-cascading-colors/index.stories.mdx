## Here are 2 blocks with different color schemes:

One is `<Block textcolor="light" bgcolor="purple">`, the other is `<Block textcolor="purple" bgcolor="light">`. Child components can utilize the ancestral text and background color. You don't have to specify the color of each element. The element's css-in-js stylesheet chooses to apply or ignore the parent color.

Other styling systems only support setting a light/dark scheme once on the entire web page. But what if your app has a dark sidebar with light text while the main content is light background with dark text? What if you have several different sections like this? You'd have to style each individual element. This library allows styling sections with different color schemes more easily. Not just light/dark, but any background/text color combinations.  


<br />

<Block as="h2" textgradient="rainbow">Putting the "C" back into "CSS-in-JS"...</Block>

Colors are managed in `colors.css`. So, they can cascade. All other variables are managed in `theme.ts`.   

Set colors on any component or HTML element by passing `data-textcolor` or `data-bgcolor` attributes.  
Make it a gradient by passing `data-textgradient` or `data-bggradient`.

A stylesheet (colors.css) sets CSS properties based on the dataset attributes. It works the same way as using classNames. CSS colors are managed in a CSS stylesheet, not in a JS theme object. Because CSS naturally cascades, all children elements are able to use the set CSS variables to apply text color, background, border, gradient, etc. This is not possible with pure CSS-in-JS.
